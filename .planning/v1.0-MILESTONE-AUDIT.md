## Cross-Phase Integration Findings

### WARNING: Convert arm skips ~/.claude/ waterfall tier for diagram paths (SCAF-03)

**Files:** [src/lib.rs](src/lib.rs) lines 213–227, [src/config.rs](src/config.rs) lines 24–37
**Requirement:** SCAF-03

The `convert` arm builds `DiagramConfig` manually (calling `dotenvy::dotenv()` + `std::env::var` directly) rather than routing through `Config::load()`. This implements only two of three waterfall tiers:

- Tier 1 — CLI flags (`--plantuml-path`, `--mermaid-path`): **applied** correctly
- Tier 2 — env vars (`PLANTUML_PATH`, `MERMAID_PATH`): **applied** correctly
- Tier 3 — `~/.claude/settings.json` fallback: **skipped**

The `update` and `upload` arms correctly call `Config::load()` which resolves all three tiers. A user whose diagram paths are stored only in `~/.claude/` will receive defaults (`"plantuml"` / `"mmdc"`) when using `convert`. The integration test `test_convert_with_diagram_path_flags` exercises tier 1 only, so this gap passes CI undetected.

### INFO: DiagramConfig::from_env() is now orphaned public API (CONV-03, CONV-04)

**File:** [src/config.rs](src/config.rs) lines 24–37

After Phase 08 integrated `DiagramConfig` into `Config::load()`, `from_env()` has no production callers — only called in unit tests within `src/converter/diagrams.rs`. The method is still `pub` and could mislead future contributors into bypassing the waterfall. Non-blocking.

### INFO: CONV-05 Converter trait unused in production integration path

**File:** [src/lib.rs](src/lib.rs)

`lib.rs` uses `MarkdownConverter` as a concrete type rather than `dyn Converter`. The `Converter` trait is validated in unit tests but the end-to-end integration path never routes through the trait abstraction. The trait still provides testability value. Non-blocking.

**Status:** Code complete, happy path untested

Gap: No passing integration test covers the full pipeline. `test_update_command_missing_api_key` only reaches credential validation — it never reaches `get_page`, `merge`, or `update_page_with_retry`. The LLM merge + Confluence round-trip path has zero automated integration test coverage.

**Status:** Code complete, happy path untested
