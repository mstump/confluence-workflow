```markdown
---
name: confluence-update
description: Update a Confluence page from a local Markdown file using the full merge pipeline (preserves inline comments). Use when the user wants to sync a .md file to Confluence.
argument-hint: "<markdown_path> <confluence_page_url>"
disable-model-invocation: true
allowed-tools: Bash(confluence-agent *)
---

Update a Confluence page from a Markdown file, preserving existing inline comments.

Run the following command and show me the output:

```bash
confluence-agent update "$0" "$1" --output json
```

Report the result to the user:
- On success: show the page URL and the number of comments kept/dropped
- On failure: show the error message and suggest checking credentials (CONFLUENCE_API_TOKEN, ANTHROPIC_API_KEY)
```

Key requirements:
- `disable-model-invocation: true` prevents Claude from auto-invoking this side-effectful command
- `allowed-tools: Bash(confluence-agent *)` scopes shell access to only the confluence-agent binary
- `"$0"` and `"$1"` are positional arguments from the user's input after `/confluence-update`
- `--output json` enables machine-readable output so Claude can parse and format the response

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code -> confluence-agent binary | Skill instructs Claude to shell out to the binary |
| User input ($0, $1) -> shell command | Positional arguments passed to shell command |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | Tampering | allowed-tools scope | mitigate | `allowed-tools: Bash(confluence-agent *)` restricts shell access to only the confluence-agent binary; blanket `Bash` is NOT used |
| T-05-04 | Elevation of Privilege | disable-model-invocation | mitigate | `disable-model-invocation: true` prevents Claude from auto-triggering side-effectful Confluence updates without explicit user intent |
| T-05-05 | Injection | $0/$1 positional arguments | accept | Arguments are passed through Claude Code's argument handling; the binary itself validates inputs (URL parsing, file path validation). Shell injection via skill arguments is mitigated by Claude Code's argument quoting. Low risk for a CLI tool invoked by the tool's own user. |
</threat_model>
