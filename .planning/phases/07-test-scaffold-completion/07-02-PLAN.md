<!-- From src/config.rs -- the two tests that need #[serial]: -->

The tests module in src/config.rs contains 10 tests. Only two call std::env::set_var
for CONFLUENCE_* keys and need serialization:

1. test_fallthrough_to_env_vars -- sets CONFLUENCE_URL, CONFLUENCE_USERNAME, CONFLUENCE_API_TOKEN
2. test_env_vars_used_when_cli_absent -- sets the same three env vars

Both tests follow this pattern:
```rust
#[test]
fn test_fallthrough_to_env_vars() {
    std::env::set_var("CONFLUENCE_URL", "https://via-env.atlassian.net");
    // ... test body ...
    std::env::remove_var("CONFLUENCE_URL");
    // ...
}
```

The fix: add `use serial_test::serial;` to the tests module imports, then add `#[serial]`
attribute after `#[test]` on both functions.
</interfaces>
