<objective>
Extract ac:inline-comment-marker elements from Confluence storage XML and classify comments as deterministic KEEP or DROP based on heading-scoped section comparison between old and new content.

Purpose: This is the foundation for the merge engine -- markers must be extracted and sections must be compared before any LLM calls happen. Deterministic short-circuits eliminate the majority of LLM calls (most comments are in unchanged sections).

Output: `src/merge/` module with extractor.rs (marker extraction), matcher.rs (section extraction + short-circuit logic), and mod.rs (shared types).
</objective>

Create src/merge/matcher.rs with:

1. `Section` struct:

```rust
#[derive(Debug, Clone, PartialEq)]
pub struct Section {
    pub heading: String,       // heading text (empty for preamble before first heading)
    pub heading_level: u8,     // 1-6 (0 for preamble)
    pub content: String,       // full HTML from heading tag to next heading
    pub start_offset: usize,   // byte offset in original content
    pub end_offset: usize,
}
```

1. `pub fn extract_sections(html: &str) -> Vec<Section>`:

- Regex: `<h([1-6])\b[^>]*>(.*?)</h\1>` to find headings
- Content before first heading becomes a preamble Section with heading="" and level=0
- Each heading's section extends from that heading tag to the start of the next heading (or end of document)
- Returns sections in document order

1. `pub fn find_matching_section<'a>(heading: &str, sections: &'a [Section]) -> Option<&'a Section>`:

- Exact string match on `section.heading` (case-sensitive)
- Returns first match

1. `pub fn strip_markers(content: &str) -> String`:

- Remove `<ac:inline-comment-marker ...>` open tags and `</ac:inline-comment-marker>` close tags
- Preserve the anchor text between paired tags
- Remove self-closing `<ac:inline-comment-marker .../>` entirely
- Uses regex: `<ac:inline-comment-marker\b[^>]*?>` for open tags, `</ac:inline-comment-marker>` for close tags, `<ac:inline-comment-marker\b[^>]*?/>` for self-closing

1. `pub fn classify_comment(marker: &CommentMarker, old_sections: &[Section], new_sections: &[Section]) -> Option<CommentDecision>`:

- Find which old section contains the marker (by position: marker.position >= section.start_offset && marker.position < section.end_offset)
- If old section not found, return Some(Drop) (orphaned marker)
- Find matching new section by heading text via find_matching_section
- If no matching new section: return Some(Drop) (section deleted -- MERGE-02 DROP short-circuit)
- Strip markers from both old and new section content, then compare with string equality
- If equal: return Some(Keep) (unchanged section -- MERGE-02 KEEP short-circuit)
- If different: return None (ambiguous -- needs LLM evaluation, MERGE-03)
