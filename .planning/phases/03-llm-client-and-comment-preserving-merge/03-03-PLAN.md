```rust
/// Merge new content into a Confluence page, preserving surviving inline comments.
///
/// This is the core merge pipeline:
/// 1. Extract comment markers from old content
/// 2. Extract sections from both old and new content
/// 3. Classify each comment: deterministic KEEP/DROP or ambiguous
/// 4. Fan out ambiguous comments to LLM with bounded concurrency
/// 5. Re-inject surviving markers into new content
pub async fn merge(
    old_content: &str,
    new_content: &str,
    llm_client: Arc<dyn LlmClient>,
    concurrency_limit: usize,
) -> Result<MergeResult, MergeError> {
```

Implementation:

- **MERGE-06 short-circuit**: If old_content is empty, whitespace-only, or equals `"<p/>"`, return new_content as-is with MergeResult { kept: 0, dropped: 0, llm_evaluated: 0 }
- **MERGE-06 short-circuit**: If extract_markers(old_content) returns empty Vec, return new_content as-is
- Extract sections from old and new content via matcher::extract_sections
- For each marker, call matcher::classify_comment:
  - Some(Keep) -> add to keep_list
  - Some(Drop) -> add to drop_list
  - None -> add to ambiguous_list with the old section and new section references
- For ambiguous markers, fan out with bounded concurrency:

  ```rust
  let semaphore = Arc::new(Semaphore::new(concurrency_limit));
  let mut handles = Vec::new();
  for (marker, old_section, new_section) in &ambiguous_list {
      let sem = semaphore.clone();
      let client = llm_client.clone();
      let marker = marker.clone();
      let old_sec = old_section.content.clone();
      let new_sec = new_section.map(|s| s.content.clone());
      let handle = tokio::spawn(async move {
          let _permit = sem.acquire().await.unwrap();
          let result = client.evaluate_comment(&old_sec, new_sec.as_deref(), &marker).await;
          (marker, result)
      });
      handles.push(handle);
  }
  ```

- Collect results: on Ok(Keep) add to keep_list, on Ok(Drop) add to drop_list, on Err(_) default to keep_list + `tracing::warn!("LLM evaluation failed for comment {}, defaulting to KEEP", marker.ac_ref)`, on JoinError same default
- Call injector::inject_markers(new_content, &keep_list) for final content
- Return MergeResult with counts

```rust
/// Inject surviving comment markers back into new content XML.
///
/// Strategy (per CONTEXT.md Decision 4):
/// 1. Exact anchor text match: find anchor_text in new content, wrap with ac:inline-comment-marker
/// 2. Fallback: find matching section by heading, inject at start of first <p> in that section
/// 3. If no match at all: log warning, drop the marker (do not corrupt XML)
pub fn inject_markers(
    new_content: &str,
    markers: &[CommentMarker],
    old_sections: &[Section],
    new_sections: &[Section],
) -> String {
```

Implementation details:

- Work on a mutable String copy of new_content
- Process markers in reverse document order (by position descending) so byte offset insertions don't shift positions of later markers
- For each marker in keep_list:
  a. **Exact match**: Search new_content for `marker.anchor_text` (non-empty anchor text only). If found, replace first occurrence with `<ac:inline-comment-marker ac:ref="{marker.ac_ref}">{marker.anchor_text}</ac:inline-comment-marker>`
  b. **Section fallback**: If exact match fails, find which old section contained this marker (by position), find matching new section by heading via find_matching_section, find first `<p>` or `<p` in that section's content, inject marker at the start of the paragraph content (after `<p>` or `<p ...>` opening tag)
  c. **Drop with warning**: If neither works, `tracing::warn!("Cannot re-inject comment {} — no anchor text match or section match in new content", marker.ac_ref)` and skip
- For self-closing markers (empty anchor_text): always use section fallback strategy (no anchor text to search for)
- Track which anchor text strings have already been injected to prevent double-injection

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM response -> merge decision | LLM failure must not corrupt output; fail-safe to KEEP |
| Old page XML -> injector | Untrusted marker content injected into new XML; must not create malformed XML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Denial of Service | merge() LLM fan-out | mitigate | `tokio::sync::Semaphore::new(concurrency_limit)` bounds parallel LLM calls. Default 5, configurable via ANTHROPIC_CONCURRENCY. Prevents rate-limit thundering herd. |
| T-03-08 | Tampering | injector.rs | mitigate | inject_markers only inserts well-formed `<ac:inline-comment-marker ac:ref="uuid">text</ac:inline-comment-marker>` elements. ac_ref value comes from original Confluence page (trusted). Anchor text is the original text from the page, not user input. If injection would corrupt XML (no valid insertion point), marker is dropped with warning. |
| T-03-09 | Information Disclosure | merge() tracing | accept | Warning logs include comment ac_ref UUIDs (not sensitive) and decision outcomes. No API keys or credentials flow through the merge engine. |
</threat_model>
