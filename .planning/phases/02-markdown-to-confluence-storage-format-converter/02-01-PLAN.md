Create `src/converter/renderer.rs` implementing a `ConfluenceRenderer` struct with a `pub fn render(markdown: &str) -> (String, Vec&lt;DiagramBlock&gt;)` method. The renderer iterates pulldown-cmark events and writes Confluence storage XML to a `String` buffer.

**Parser setup:**
```rust
use pulldown_cmark::{Parser, Event, Tag, TagEnd, Options, CodeBlockKind, Alignment};
use std::fmt::Write;
```
Options: `ENABLE_TABLES | ENABLE_STRIKETHROUGH | ENABLE_YAML_STYLE_METADATA_BLOCKS | ENABLE_TASKLISTS`

**Struct fields:**
- `output: String` — the XML buffer
- `in_code_block: bool` — inside a fenced code block
- `code_language: Option&lt;String&gt;` — language of current code block
- `code_content: String` — accumulated code text
- `table_alignments: Vec&lt;Alignment&gt;` — current table column alignments
- `in_table_head: bool` — inside `<thead>`
- `first_h1_skipped: bool` — tracks whether first H1 has been skipped (Pitfall 4)
- `skip_heading_content: bool` — when true, suppress text events (used during first H1 skip)

**DiagramBlock struct** (public, in renderer.rs):
```rust
pub struct DiagramBlock {
    pub kind: String,        // "plantuml", "puml", or "mermaid"
    pub content: String,     // raw diagram source
}
```

**Event mapping (complete list):**

| Event | Confluence XML |
|-------|---------------|
| `Start(Heading { level, .. })` | `<hN>` (skip first h1, set skip_heading_content=true) |
| `End(Heading(level))` | `</hN>` (reset skip_heading_content) |
| `Start(Paragraph)` | `<p>` |
| `End(Paragraph)` | `</p>` |
| `Start(CodeBlock(Fenced(lang)))` | If lang is plantuml/puml/mermaid: set in_code_block, collect content for DiagramBlock. Otherwise: set in_code_block, store language. |
| `End(CodeBlock)` | If diagram: push DiagramBlock, write `<!-- DIAGRAM_PLACEHOLDER_N -->`. If code: emit expand macro wrapping code macro (see below). |
| `Start(BlockQuote(_))` | `<blockquote>` |
| `End(BlockQuote(_))` | `</blockquote>` |
| `Start(List(None))` | `<ul>` |
| `Start(List(Some(start)))` | `<ol>` (ignore start number; Confluence doesn't support start attr well) |
| `End(List(..))` | `</ul>` or `</ol>` based on whether ordered |
| `Start(Item)` | `<li>` |
| `End(Item)` | `</li>` |
| `Start(Table(alignments))` | Store alignments, write `<table>` |
| `End(Table)` | `</table>` |
| `Start(TableHead)` | `<thead><tr>`, set in_table_head=true |
| `End(TableHead)` | `</tr></thead>`, set in_table_head=false, write `<tbody>` |
| `Start(TableRow)` | `<tr>` |
| `End(TableRow)` | `</tr>` |
| `Start(TableCell)` | `<th>` if in_table_head, `<td>` otherwise |
| `End(TableCell)` | `</th>` or `</td>` |
| `Start(Emphasis)` | `<em>` |
| `End(Emphasis)` | `</em>` |
| `Start(Strong)` | `<strong>` |
| `End(Strong)` | `</strong>` |
| `Start(Strikethrough)` | `<span style="text-decoration: line-through;">` |
| `End(Strikethrough)` | `</span>` |
| `Start(Link { dest_url, title, .. })` | `<a href="{escaped_url}">` |
| `End(Link)` | `</a>` |
| `Start(Image { dest_url, .. })` | `<ac:image ac:alt="{alt}"><ri:attachment ri:filename="{filename}" /></ac:image>` — extract filename from dest_url path |
| `Text(text)` | If in_code_block: append to code_content. If skip_heading_content: ignore. Otherwise: XML-escape and append. |
| `Code(text)` | `<code>{xml_escaped_text}</code>` |
| `SoftBreak` | `\n` |
| `HardBreak` | `<br />` |
| `Rule` | `<hr />` |
| `MetadataBlock(..)` | Skip entirely (CONV-02 frontmatter stripping) |
| `TaskListMarker(checked)` | Emit checkbox character: checked=`[x]`, unchecked=`[ ]` |

**Code block emission** (expand macro wrapping code macro per Python converter behavior):
```rust
fn emit_code_block(&mut self) {
    self.output.push_str(r#"<ac:structured-macro ac:name="expand">"#);
    self.output.push_str(r#"<ac:parameter ac:name="title">Source</ac:parameter>"#);
    self.output.push_str("<ac:rich-text-body>");
    self.output.push_str(r#"<ac:structured-macro ac:name="code">"#);
    if let Some(ref lang) = self.code_language {
        if !lang.is_empty() {
            write!(self.output, r#"<ac:parameter ac:name="language">{}</ac:parameter>"#, lang).unwrap();
        }
    }
    self.output.push_str("<ac:plain-text-body><![CDATA[");
    // CDATA split for ]]> (Pitfall 2)
    let safe_content = self.code_content.replace("]]>", "]]]]><![CDATA[>");
    self.output.push_str(&safe_content);
    self.output.push_str("]]></ac:plain-text-body>");
    self.output.push_str("</ac:structured-macro>");
    self.output.push_str("</ac:rich-text-body>");
    self.output.push_str("</ac:structured-macro>");
    self.code_content.clear();
}
```

**XML escaping utility** (for text nodes and attribute values):
```rust
fn escape_xml(s: &str) -> String {
    s.replace('&', "&amp;")
     .replace('<', "&lt;")
     .replace('>', "&gt;")
     .replace('"', "&quot;")
     .replace('\'', "&apos;")
}

fn escape_attr(s: &str) -> String {
    // Same as escape_xml — used for attribute values
    escape_xml(s)
}
```

**List type tracking:** Maintain a `list_stack: Vec&lt;bool&gt;` (true = ordered) to correctly close `</ul>` vs `</ol>` for nested lists. Push on `Start(List(..))`, pop on `End(List(..))`.

**Table tbody tracking:** Write `</tbody>` before `</table>` only if the table had rows (i.e., TableHead was encountered).
