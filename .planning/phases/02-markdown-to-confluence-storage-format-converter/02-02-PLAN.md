<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Markdown content -> XML output | Untrusted content must be XML-escaped before embedding in storage format |
| Image paths -> ri:filename | Directory components stripped to prevent path traversal in attachment references |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | renderer.rs text output | mitigate | XML-escape all text via escape_xml() — covered in Plan 01 implementation, tested here |
| T-02-04 | Information Disclosure | renderer.rs image handling | mitigate | Strip directory components from image src; only use basename for ri:filename attribute |
| T-02-05 | Tampering | renderer.rs link href | mitigate | XML-escape href attribute values to prevent attribute injection |
</threat_model>
