|----------|-------------|
| User CLI input -> subprocess spawn | `--plantuml-path` / `--mermaid-path` values passed to `Command::new()` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | `--plantuml-path` CLI flag | accept | Path is supplied by the authenticated local user who runs the CLI binary. The user already has the same privilege level as the process. The existing `Command::new(plantuml_path)` pattern in `converter/diagrams.rs` is unchanged -- this phase only changes where the value originates (CLI flag vs env var), not how it is consumed. No additional validation needed. |
| T-08-02 | Elevation of Privilege | `--plantuml-path` pointing to malicious binary | accept | Same trust boundary as T-08-01. The user who supplies the flag can already execute arbitrary binaries. This is standard CLI behavior (comparable to `$PATH` resolution). |
</threat_model>
