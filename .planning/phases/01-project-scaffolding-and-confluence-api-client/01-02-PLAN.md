**Config struct** -- the resolved, validated configuration:

```rust
pub struct Config {
    pub confluence_url: String,      // Required for upload/update commands
    pub confluence_username: String,  // Required for upload/update commands
    pub confluence_api_token: String, // Required for upload/update commands
    pub anthropic_api_key: Option<String>, // Optional in Phase 1; required in Phase 3
}
```

**Config::load(overrides: &CliOverrides) -> Result<Self, ConfigError>** -- Waterfall logic:

1. Call `dotenvy::dotenv().ok()` to load `.env` file (non-fatal if missing)
2. For each required field (confluence_url, confluence_username, confluence_api_token):
   a. Check CLI override (from CliOverrides)
   b. Check environment variable (`CONFLUENCE_URL`, `CONFLUENCE_USERNAME`, `CONFLUENCE_API_TOKEN`)
   c. Attempt `~/.claude/` fallback: call a private helper `load_from_claude_config(key: &str) -> Option<String>` that tries to read `~/.claude/settings.json` and look for a matching key. Log a tracing::debug message if the file doesn't exist. Return None if not found.
   d. If all sources return None, return `Err(ConfigError::Missing { name: "CONFLUENCE_URL" })` (with the appropriate field name)
3. For anthropic_api_key: same waterfall but return `Option<String>` (None is OK in Phase 1). Check env var `ANTHROPIC_API_KEY`.
4. Normalize confluence_url: `url.trim_end_matches('/').to_string()` to prevent double-slash in API paths.

**Private helper `load_from_claude_config`** -- stub implementation:

- Use `dirs::home_dir()` to resolve `~`
- Try to read `~/.claude/settings.json`
- If file exists, parse as `serde_json::Value` and look for the key
- Log `tracing::debug!("~/.claude/settings.json not found, skipping")` if missing
- This is a best-effort fallback; it should never cause an error
