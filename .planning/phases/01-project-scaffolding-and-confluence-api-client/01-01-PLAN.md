Create the clap CLI definition using derive macros (per SCAF-02).

**src/cli.rs** -- Define these types:

- `#[derive(Parser)] struct Cli` with:
  - `#[command(name = "confluence-agent", about = "Convert and upload Markdown to Confluence")]`
  - `command: Commands` field
  - `#[arg(long, env = "CONFLUENCE_URL")] confluence_url: Option<String>` -- global flag for Confluence base URL
  - `#[arg(long, env = "CONFLUENCE_USERNAME")] confluence_username: Option<String>` -- global flag
  - `#[arg(long, env = "CONFLUENCE_API_TOKEN")] confluence_token: Option<String>` -- global flag
  - `#[arg(long, short)] verbose: bool` -- enable debug logging

- `#[derive(Subcommand)] enum Commands` with:
  - `Update { markdown_path: PathBuf, page_url: String }` -- "Convert markdown and merge with existing page, preserving inline comments"
  - `Upload { markdown_path: PathBuf, page_url: String }` -- "Convert markdown and overwrite page directly (no LLM merge)"
  - `Convert { markdown_path: PathBuf, output_dir: PathBuf }` -- "Convert markdown to Confluence storage XML locally"

Each subcommand variant must have a `#[command(about = "...")]` with the description above.

Update `src/lib.rs` `run()` function to match on `Commands::Update`, `Commands::Upload`, `Commands::Convert` and print a message like "update command: not yet implemented" for each, returning `Ok(())`.

Ensure `cargo build` still succeeds with zero warnings after this change.
