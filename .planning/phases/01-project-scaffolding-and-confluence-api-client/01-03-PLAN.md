**src/confluence/client.rs** -- Full production implementation:

**ConfluenceClient struct:**
- Fields: `client: reqwest::Client`, `base_url: String`, `auth_header: String`
- Constructor `new(base_url: &str, username: &str, api_token: &str) -> Self`:
  - Build reqwest::Client with 30s timeout
  - Base64-encode `username:api_token` for Basic Auth header using `base64::engine::general_purpose::STANDARD`
  - Normalize base_url: `base_url.trim_end_matches('/')` (pitfall 4 from research)

**Implement ConfluenceApi for ConfluenceClient:**

`get_page(page_id: &str)`:
- GET `{base_url}/rest/api/content/{page_id}?expand=body.storage,version`
- Set `Authorization` header
- Match response status: 200 -> deserialize Page, 401 -> Unauthorized, 404 -> PageNotFound, other -> UnexpectedStatus
- Use `tracing::debug!` to log the request URL

`update_page(page_id: &str, title: &str, content: &str, version: u32)`:
- PUT `{base_url}/rest/api/content/{page_id}`
- JSON body: `{ version: { number: version, minorEdit: true }, title, type: "page", body: { storage: { value: content, representation: "storage" } } }`
- Set `Authorization` and `Content-Type: application/json` headers
- Match response status: 200 -> Ok, 409 -> VersionConflict, 401 -> Unauthorized, 404 -> PageNotFound, other -> UnexpectedStatus
- Use `tracing::info!` to log successful update with page_id and version

`upload_attachment(page_id: &str, filename: &str, content: Vec<u8>, content_type: &str)`:
- POST `{base_url}/rest/api/content/{page_id}/child/attachment`
- Headers: `Authorization`, `X-Atlassian-Token: nocheck` (CRITICAL -- without this, Confluence returns 403)
- Multipart form: `file` part with the content bytes, filename, and content_type; `minorEdit` text field = "true"
- Check response `is_success()` or return AttachmentUpload error

**Free function `update_page_with_retry`** (per CONF-02 TOCTOU mitigation):
```rust
pub async fn update_page_with_retry(
    client: &dyn ConfluenceApi,
    page_id: &str,
    content: &str,
    max_retries: u32,  // default: 3
) -> Result<(), ConfluenceError>
```
- Loop up to `max_retries + 1` attempts:
  1. Fetch page with `client.get_page(page_id)` to get current version and title
  2. Call `client.update_page(page_id, &page.title, content, page.version.number + 1)`
  3. On success, return Ok
  4. On `ConfluenceError::VersionConflict` and attempts remaining: log warning with `tracing::warn!`, continue loop
  5. On any other error: return immediately
- After exhausting retries, return the last VersionConflict error

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-01-05 | T | ConfluenceClient auth | mitigate | Use HTTPS only (reqwest with rustls); Basic Auth header over TLS is standard for Confluence Cloud API |
| T-01-06 | S | extract_page_id | mitigate | Regex extracts only digits (\d+); no code injection possible. Validate extracted ID is non-empty. |
| T-01-07 | T | update_page | mitigate | Version conflict detection (409) prevents overwriting concurrent edits; retry-with-re-fetch is the standard pattern |
| T-01-08 | D | Confluence API | accept | API availability is outside our control; reqwest timeout (30s) prevents hanging. Typed errors surface failures clearly. |
